# Boas Práticas de Colaboração

- **Commits curtos e claros**: Use mensagens descritivas e faça commits pequenos e focados
- **Branches temáticas**: Crie uma branch por funcionalidade ou correção
- **Pull Requests bem documentados**: Descreva o que foi alterado e por quê
- **Revise o código**: Participe de code reviews de forma construtiva
- **Comunique-se**: Use issues e discussões para alinhar com o time
- **Sincronize frequentemente**: Faça pull da main branch regularmente
- **Respeite convenções**: Siga padrões de código do projeto
- **Testes**: Garanta que seu código está testado antes de enviar
